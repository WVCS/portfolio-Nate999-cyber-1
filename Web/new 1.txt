  <h1> HTML Postions and Examples </h1>
  <p> A website explaining the different properties of HTML code. </p>
 <div class="container1">
    <h2 class="header"> Postion Static </h2>
    <p2 class="text"> HTML elements are positioned static by default.Static positioned elements are not affected by the top, bottom, left, and right properties.An element with position: static; is not positioned in any special way; it is always positioned according to the normal flow of the page:</p2>
<div class="container2">
    <h2 class="header"> Postion Relative </h2>
    <p3 class="text"> An element with position: relative; is positioned relative to its normal position. Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.</p23>
<div class="container3">
    <h2 class="header"> Postion Fixed </h2>
    <p4 class="text"> An element with position: fixed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element.A fixed element does not leave a gap in the page where it would normally have been located. ghhghghgfgfkj.fiofgoijfgoi;ufgo8;g;yhyhgiugyg;8ryhvviaurgynyvhc svrgbghnrjlkghljhnvjsgvhufuh e</p34>
<div class="container4">
 		<h2 class="header"> Postion absolute </h2>
  	<p5 class="text"> An element with position: absolute; is positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed).However; if an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling. </p5>
<div class="container5">
 		<h2 class="header"> Postion Sticky </h2>
  	<p6 class="text"> An element with position: sticky; is positioned based on the user's scroll position.A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position:fixed)</p6>
